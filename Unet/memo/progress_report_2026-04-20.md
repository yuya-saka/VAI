# 進捗報告：頸椎 CT 自動解析（2026-04-20）

---

## 前回からの修正：GT ライン抽出の根本バグ解消

研究初期、**GT（教師データ）の線方向の計算に根本的なバグが存在していた**。この修正により精度が大幅に改善した。本報告の全実験はこれらの修正適用済みである。

### 修正 1：GT 角度計算（端点法 → PCA 法）

**問題**: アノテーションのポリラインが V 字型（折れ線）の場合、先頭・末尾の端点だけで角度を計算すると、ポリラインの主軸方向とは全く異なる角度が得られる。

![ライン検出（C3）](images/sample1_C3_slice055_comparison.png)

| 状況 | 数値 |
|---|---|
| V 字型ポリラインの割合 | 全データの **57.3%**（3639 本中） |
| 端点法で角度誤差 >10° の割合 | **46.4%**（ほぼランダム） |

**対応**: ポリライン全点に対して PCA（主成分分析）で主軸を求める方式に変更。

| | angle error [°] | rho error [px] | perp dist [px] |
|---|:---:|:---:|:---:|
| 修正前（端点法） | 28.19 | 9.87 | 18.61 |
| **修正後（PCA 法）** | **5.27** | **3.01** | **10.76** |

→ **angle error が 28° → 5° に激減**。この修正が精度改善の最大の要因。

## 概要

頸椎 CT（軸位断）の C1〜C7 スライスに対して、**4領域セグメンテーション**と**椎体境界ライン検出**を UNet 系モデルで実装・評価した。

**データ**: C1〜C7 スライス 906 枚、5-fold CV

**アノテーション種別**:

| 種別 | 内容 |
|---|---|
| 折れ線（ポリライン） | 各椎体の境界線 4 本 |
| 4 領域マスク | 椎体 / 右椎弓根 / 左椎弓根 / 後弓 |

### 線アノテーション → マスク自動生成

本研究の重要な基盤として、**折れ線アノテーションさえあれば 4 領域マスクを自動生成できる**仕組みを実装した。

**生成の流れ**:

| ① CT 画像 | ② 折れ線アノテーション | ③ 自動生成マスク |
|:---:|:---:|:---:|
| ![CT](images/dataset_C3_ct.png) | ![折れ線](images/dataset_C3_overlay.png) | ![マスク](images/dataset_C3_gt_overlay.png) |
| 入力 CT（軸位断） | 4 本の折れ線を引くだけ | 椎体（赤）/ 右椎弓根（青）/ 左椎弓根（緑）/ 後弓（黄）|

**仕組み**: 各折れ線を境界として画像を半平面に分割し、4本の線の組み合わせで 4 領域を定義する。アノテーターはマスクを手動で塗る必要がなく、**折れ線のみでラベリングが完結する**。

## 1. 使用モデル

### 1.1 line_only（TinyUNet）

- **構成**: 軽量 UNet（TinyUNet）
- **入力**: CT 画像（1ch）
- **出力**: 4ch ガウシアンヒートマップ（境界線 1 本につき 1ch）
- **推論**: ヒートマップ → モーメント法 → 法線形式（φ, ρ）でライン推定
- **学習**: ヒートマップ MSE のみ（σ=3.5）

### 1.2 seg_only（ResUNet）

- **構成**: ResBlock 付き UNet（ResUNet, ~1.6M パラメータ）
- **入力**: CT 画像（1ch）
- **出力**: 5ch セグメンテーション（background + 4 領域）
- **学習**: Cross-entropy + Dice 損失

### 1.3 multitask（ResUNet, 共有エンコーダ）

- **構成**: seg_only と同じ ResUNet に、line 用デコーダを追加

```
Encoder（共有）
    ├── Decoder_seg → 5ch セグメンテーション
    └── Decoder_line → 4ch ライン ヒートマップ
```

- **学習**: `L_total = L_line_MSE + α_seg · L_seg`
- **チューニングの核心**: `α_seg`（seg 損失の重み）の調整
  - α が大きすぎると seg 勾配が line を圧迫（negative transfer）
  - α が小さすぎると seg 精度が向上しない
  - **α=0.07 は `L_line_MSE` と `α_seg · L_seg` のスケールがほぼ等しくなるよう調整した値**（両損失を均等に扱う設定）

**試みたバリアント**:

| バリアント | 変更内容 | 効果 |
|---|---|---|
| α_seg sweep | 0.02 / 0.07 / 0.15 を比較 | α=0.02 がライン最良、α=0.07 が seg 最良 |
| 椎体条件付け | C1〜C7 の one-hot ラベルをエンコーダに注入 | C1/C2 の angle ▼1.1〜1.4° |
| Shared decoder | DecoderTrunk を seg/line で共有、後半のみ分岐 | 効果は限定的 |
| データ拡張修正 | heatmap 補間 → ポリライン変換後に再生成 | angle/rho が改善 |

## 4. multitask vs seg_only 比較

### セグメンテーション精度（paired t-test, n=5 folds, vs seg_only_v2 baseline）

| モデル | Δ mIoU | p 値 | fold 勝ち数 |
|---|:---:|:---:|:---:|
| seg_only + 境界損失 | -0.0004 | 0.926 | 1/5 |
| multitask α=0.02 | -0.0003 | 0.950 | 1/5 |
| **multitask α=0.07** | **+0.0032** | 0.480 | 2/5 |
| multitask α=0.15 | -0.0018 | 0.660 | 2/5 |

→ **いずれも統計的有意差なし**（n=5 の限界）

→ multitask α=0.07 のみが一貫して正方向だが、p=0.48 にとどまる

| α_seg | seg mIoU | angle error [°] | 傾向 |
|:---:|:---:|:---:|---|

## 5. まとめ

### できたこと

| 項目 | 内容 |
|---|---|
| 線 → マスク自動生成 | 折れ線アノテーションから4領域マスクを自動構築。アノテーションコスト削減に有効 |
| 4 領域セグメンテーション | mIoU 0.905 前後で安定動作 |
| ライン検出 | 平均 angle error 5.5°（中位頸椎は 4° 以下） |

### 限界・課題

| 項目 | 内容 |
|---|---|
| multitask の seg 改善は限定的 | 最良でも Δ mIoU = +0.003、有意差なし |
| C1・C2 のライン精度 | angle error 7〜8° と他椎体の倍近く。形状特異性が原因と推定 |
| 外れ値の存在 | 特定患者で 90° 近い angle error が連続して発生。モデルの robustness が課題 |

### 考察

line heatmap（σ=3.5）は線近傍の数ピクセルにしか勾配信号がない。  
そのため共有エンコーダ経由でも、seg デコーダ全体への情報伝搬が限定的となり、  
「ライン情報でセグメンテーション境界を改善する」という仮説を実証するには至らなかった。

### 今後の方向性（検討中）

**Distance Map 回帰**を補助タスクに採用する方針。

- 「ここが線」（heatmap, 線近傍のみ）→「各ピクセルと境界の距離」（distance map, 全ピクセル）
- 全ピクセルに監視信号が届く → seg デコーダへの情報伝搬が改善する可能性
- 符号付き距離（SDF）で境界内外を区別
