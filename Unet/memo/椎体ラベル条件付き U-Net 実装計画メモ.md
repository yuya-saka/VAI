# 椎体ラベル条件付き U-Net 実装計画メモ

## 1. 目的

現在の U-Net は、CT画像 + 椎体マスクの2チャネル入力から、4本の境界線ヒートマップを回帰する構成である。  
しかし、頸椎は C1〜C7 で解剖学的形状が大きく異なるため、画像だけから境界位置や角度を一律に学習させると、椎体ごとの差異が混在して学習が不安定になる可能性がある。

そこで、各サンプルがどの椎体（C1〜C7）に対応するかを示す `vertebra` 情報をモデルに入力し、**椎体レベルを条件とした境界推定**を行う。  
最初の段階では、実装変更を最小限に抑えるため、**椎体ラベルを one-hot ベクトル化して U-Net の bottleneck に付加する方法**を採用する。

---

## 2. 最初に試す実装方針

### 方針
- 入力画像は従来通り
  - CT画像
  - 椎体マスク
- 出力も従来通り
  - 4チャネルの境界線ヒートマップ
- 追加情報として
  - `vertebra`（C1〜C7）を整数ID化
  - one-hot ベクトルに変換
  - bottleneck feature に結合

### この方法を最初に採用する理由
- 既存コードへの変更が小さい
- ベースラインとの差分が明確
- データ数が少ない段階でも比較的安定
- 効果検証後に embedding や multi-task へ拡張しやすい

---

## 3. 実装概要

## 3.1 椎体ラベルの表現

各椎体を以下のように整数IDへ変換する。

- C1 -> 0
- C2 -> 1
- C3 -> 2
- C4 -> 3
- C5 -> 4
- C6 -> 5
- C7 -> 6

これを 7 次元 one-hot ベクトルへ変換する。

例:
- C1 -> `[1,0,0,0,0,0,0]`
- C3 -> `[0,0,1,0,0,0,0]`
- C7 -> `[0,0,0,0,0,0,1]`

---

## 3.2 モデルへの組み込み位置

最初は **U-Net encoder の最終出力（bottleneck）にのみ付加**する。

### 理由
- 浅い層はエッジや濃淡など低レベル特徴を扱うため、椎体ラベルを直接入れる必然性が低い
- 深い層では「このスライスは C2 である」「これは C6 である」という文脈情報が有効
- モデル変更量を最小限にできる

---

## 3.3 結合方法

encoder の最終特徴を

- `bottleneck`: `[B, C, Hb, Wb]`

とする。

one-hot ベクトル `[B, 7]` を空間方向へ展開し、

- `[B, 7, Hb, Wb]`

に変換して、チャネル方向に結合する。

- `concat([bottleneck, onehot_map], dim=1)`

その後、1x1 convolution でチャネル数を整えて decoder に入力する。
