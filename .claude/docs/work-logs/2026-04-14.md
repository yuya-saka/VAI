# 作業ログ 2026-04-14

## 実施内容

### 1. 進捗振り返り・方針転換の決定

**結論**: line heatmap ベースの multitask 改善は打ち止め。distance map 回帰への方向転換を決定。

**これまでの試行と結果**:
- multitask_v1 (α=0.02): angle 5.30° / mIoU 0.909 — seg_only (0.906) からの改善は微小
- multitask_v2 (椎体条件付け): C1/C2 の angle は改善したが seg は微減 (0.907)
- shared decoder 実装済みだが、heatmap の監視信号が線近傍に限定される根本問題は構造変更だけでは解決しない

**問題の本質**:
- line heatmap (σ=3.5) は線近傍の数ピクセルにしか勾配信号がない
- seg decoder への情報伝搬が限定的で、劇的な seg 精度向上に繋がらなかった

### 2. 新方針: distance map 回帰を補助タスクに

**コンセプト**:
- 「ここが線そのものです」(heatmap) → 「各ピクセルが境界線からどれくらい離れているか」(distance map)
- 全ピクセルに監視信号が行き渡る
- 距離が急変する場所＝境界 → seg boundary sharpening に直結

**次セッションで詰める設計ポイント**:
1. 距離の定義 — ユークリッド距離 or 符号付き距離 (±)
2. タスク構成 — `L_seg + β·L_dist` (seg の補助タスク) or line heatmap を完全置換
3. GT 生成方法 — 既存折れ線アノテーションから `distance_transform_edt` 等で生成

**次セッションの流れ**: Codex に設計相談 → 方針決定 → 実装
