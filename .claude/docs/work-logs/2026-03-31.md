### 2. 異方性ガウシアン提案の検討（却下）

提案内容: GT ヒートマップを等方性 → 異方性ガウシアンに変更

**Codex 分析結果（`.claude/docs/codex/20260331-anisotropic-gaussian-evaluation.md`）:**

- **前提が誤り**。距離変換ベースの現 GT は「線への垂直距離」を使うため、既に自然な異方性を持つ（λ_parallel ≈ L²/12, λ_perp ≈ σ²）
- 異方性 Gaussian への変更は「新情報の追加」ではなく「勾配配分の変更」にすぎない
- 角度誤差の本当の原因は GT 形状ではなく：
  1. MSE 損失が角度を直接最適化しない（目標ミスマッチ）
  2. 推定器のロバスト性不足（閾値依存、crosstalk、境界切れ）
  3. データ規模（~150例）とアノテーションばらつき

**→ 採用しない。geometry loss の有効化を優先する。**
