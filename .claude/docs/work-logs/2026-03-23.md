# 2026-03-23: Threshold Processing Investigation

## 問題分析

**テスト結果（Y軸修正適用済み）:**
- angle_error: 32.70°（まだ高い、<10°が期待値）
- Confidence: 0.214（非常に低い、線品質が低いことを示す）

**根本原因の仮説:**
- 低confidence → 不安定な角度計算
- mu20 ≈ mu02 のとき `atan2(2*mu11, mu20-mu02)` の分母が小さくなる
- ヒートマップに低強度の「テール」が含まれており、等方的に分散を広げる

## Threshold処理実験

**実験設定:** threshold=0.0（なし）vs threshold=0.2

**結果（411テストサンプル）:**

| メトリクス | Thresholdなし | Threshold≥0.2 | 改善 |
|-----------|-------------|--------------|------|
| 平均誤差 | 32.70° | **23.73°** | -8.97°（27.4%） |
| 中央値誤差 | 28.10° | **10.09°** | -18.01°（64.1%） |
| 最大誤差 | 89.61° | 89.49° | ほぼ同じ |
| Confidence | 0.214 | **0.941** | +0.726（340%） |

**主な知見:**
- 98.8% のサンプルで改善または変化なし
- Confidenceが劇的に向上（0.214 → 0.941）
- 1.2% のサンプルで悪化（411中5件）
  - 悪化したサンプル: 元々ヒートマップ応答が低い
  - Thresholdが重要な低強度領域をカットしてしまった

## なぜ効果があるか

1. 低強度ノイズを除去 → クリーンなヒートマップ
2. mu20 ≈ mu02 のケースを削減 → atan2が安定
3. λ1/λ2比率が向上 → confidenceが高くなる

**トレードオフ:** 98.8%が改善するが1.2%が悪化（許容範囲）。悪化ケースは弱いヒートマップ応答（thresholdではなく訓練の問題）。

## 重要な洞察

**角度誤差は座標バグや数式の誤りが原因ではない。**
**ノイズが多い等方的ヒートマップから角度計算する際の数学的不安定性が原因。**
**Threshold処理は効果的な後処理ソリューション。**
