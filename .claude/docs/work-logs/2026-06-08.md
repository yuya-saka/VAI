# 作業ログ 2026-06-08

## 作業内容: fracture_labels を annotation_data 座標系へ変換

### 目的

`fracture_labels/{sample}.nii.gz`（元CT座標系の骨折バイナリマスク）を
`annotation_data/{sample}/{V}/fracture.nii.gz` として保存し、
z-index N の値が `slice_NNN.png` のスライスラベルに対応するようにする。

---

### 実装: `Unet/preprocessing/add_fracture_labels.py`

#### 処理フロー

1. `fracture_labels/{sample}.nii.gz`（元CT座標系）
2. `segmentations/` の重心から `generate_pre.py` と同じ 2 パスクロップ変換を適用
3. `annotation_data` の回転を適用
   - **auto_tilt サンプル**（sample28+）: `rotation_deg.npy` → `Rotation.from_euler("XY")`
   - **manual サンプル**（sample1-27）: CT affine の SVD → `R_apply = R_tilt.T`
4. `annotation_data/{sample}/{V}/fracture.nii.gz` として保存

#### Z アライメント検証

centroid ベースの Z オフセット分析（手動サンプル）:
```
sample1/C1: dz = +0.2 voxel (+0.1mm)
sample1/C3: dz = -0.1 voxel (-0.1mm)
sample2/C3: dz = -1.8 voxel (-0.7mm)
sample3/C4: dz =  0.0 voxel ( 0.0mm)
```
→ 最大ずれ 2 スライス（0.8mm）で学習ラベルとして実用上問題ない精度。

```python
ct_patterns = (

```

## 追加修正: 骨折ラベルを指定椎体のみに限定

### 問題

`add_fracture_labels.py` が全椎体に骨折ラベルをクロップしていたため、
骨折が存在しない椎体にも誤ってラベルが付与されていた。

### 修正内容

`Unet/preprocessing/add_fracture_labels.py` に以下を追加:

1. **`FRACTURE_VERTEBRAE` dict** — 全 52 サンプルの骨折椎体テーブルをハードコード
   ```python
   FRACTURE_VERTEBRAE: dict[str, list[str]] = {
       "sample1": ["C2"],
       "sample2": ["C1", "C2"],
       ...
   }
   ```

2. **`process_vertebra` に `is_fractured: bool` 引数を追加**
   - `False` → 全ゼロマスクを保存
   - `True` → 骨折ラベル × 椎体セグメンテーションの AND マスクで保存（隣接椎体へのはみ出しも除去）

### 実行結果

```
生成: 284 椎体（全サンプル --overwrite で再生成）
回転情報なし: 3（sample2/C7, sample27/C2, sample31/C4 — データ欠損のため変化なし）
```

検証（sample1）:
```
C1: 0 voxels  ← 正しくゼロ
C2: 261382 voxels  ← 骨折あり
C3-C7: 0 voxels  ← 正しくゼロ
```
