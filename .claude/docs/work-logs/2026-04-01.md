## 実施内容

### 1. eval_error_viz.py の新規実装

`Unet/debug/eval_error_viz.py`（940行）を実装。
全fold の `_PRED_lines.json` を集計して誤差分布・worst sample を可視化するスクリプト。

**主要機能:**

| 機能 | 内容 |
|------|------|
| 全体ヒストグラム | angle/rho/perp の percentile縦線付き分布 |
| violin+box plot | 椎体別（C1-C7）・line別（line_1-4）の分布 |
| Bland-Altman | angle（φ、180°周期符号付き差分）・rho（ピクセル換算）の ±1.96SD プロット |
| worst sample 個別PNG | 予測ヒートマップ＋ライン比較の2行構成タイル |
| worst summary grid | angle/rho 両メトリクス × top-20 件のグリッド画像 |

**使い方:**
```bash
uv run python Unet/debug/eval_error_viz.py \
    --exp-dir Unet/outputs/しきい値変更/0.6_sig4.0 \
    --top-n 20 \
    --metric angle rho
```

**出力構造:**
```
{exp-dir}/vis/error_viz/
├── line_records.csv / slice_records.csv / percentiles.csv / summary.txt
├── plots/
│   ├── angle_overall_hist.png
│   ├── angle_by_vertebra_violin.png
│   ├── angle_by_line_violin.png
│   ├── rho_overall_hist.png / rho_by_vertebra_violin.png / ...
│   ├── bland_altman_angle.png
│   └── bland_altman_rho.png
└── worst_samples/
    ├── angle_max/ worst_summary_grid.png + rank*.png
    └── rho_max/   worst_summary_grid.png + rank*.png
```

**各タイルの構成（worst個別PNG）:**
- 上段: `heatmap_lines.png`（予測ヒートマップ | 予測線 | GT線）
- 下段: `comparison.png`（GT線 | 予測線）

**備考:**
- GTヒートマップは trainer が test set 全件に保存していないため非対応
  - `save_examples(n_save=16)` でval/test先頭16件のみ保存される設計
  - 必要なら trainer.py の `predict_lines_and_eval_test` 内で GT_merged も保存するよう変更が必要
