## RSNA line-region QC: 幾何例外の確認

### 背景

`Unet/outputs/rsna_line_predictions` から、`rsna_line_vis` と同じ短線分方式で4領域分割を再生成し、全症例について以下を確認した。

- 4領域の面積異常
- 入力CTと椎骨maskのalignment
- orientation correction（向き補正）の異常候補

重要な確認:

- `rsna_line_vis` は JSON の raw `endpoints` をそのまま使っていない。
- 正しい方式は、JSONの `centroid + angle_deg` に、学習データから計算したline別平均線長を掛けて短い線分を復元する方法。
- 代表例 `1.2.826.0.1.3680043.10001/C3_plane06` で、この復元方式が保存済み `rsna_line_vis` の4 Regionsパネルとピクセル完全一致することを確認した。
- raw `endpoints` を使った初回QC結果は無効。正しい方式で再集計済み。

#### mask反転候補

CT edge / intensity alignment によるheuristicで、以下2件を検出した。

### 決定

以下studyは使用しない。

- `1.2.826.0.1.3680043.24673`

理由:

- DICOM series内でslice orientationが連続的に変わる既知の例外症例。
- forced merge / affine repair を通っている。
- CT samplingはDICOM geometry、mask samplingはrepair済みNIfTI affineを使うため、両者の物理座標整合が崩れる。
- level/planeによって、反転・回転・平行移動のようなズレが出る。
- bboxを持つC7にも影響しているため、後段で補正して使うのは危険。

以下studyは使用継続可。

- `1.2.826.0.1.3680043.10001`
- `1.2.826.0.1.3680043.17382`
