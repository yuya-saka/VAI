# 作業ログ 2026-06-19

## bbox対象失敗症例の修復

対象:

- `1.2.826.0.1.3680043.24673`
- 骨折bbox: 17件
- 骨折level: C4、C7

原因:

- 218枚の同一DICOM series内で`ImageOrientationPatient`が連続的に変化
- 通常の`dcm2niix`変換が150個のNIfTI候補へ分割
- `dcm2niix -m y`で単一volume化できるが、slice軸affineが0

対応:

- 複数NIfTI候補が生成された場合のみ`dcm2niix -m y`で再試行
- merge出力のslice軸が0の場合、DICOM物理位置の両端から平均RAS slice vectorを補完
- bbox mappingでは補完affineを正本にせず、従来どおり各DICOM sliceのpatient coordinateを使用する

結果:

- TotalSegmentator処理時間: 約93秒
- C1-C7 mask: 7件すべて生成
- mask shape: `(512, 512, 218)`
- 全mask: non-empty
- 全mask: affine一致
- bbox: 17件を確認
- segmentation完了症例: `2016 / 2019`
- 残り3件はユーザー方針により未修復
