## 2. 椎体骨折検出ベースライン 設計確定（4領域mask不使用・新方向）

草案 (`memo/草案.md`) の Step1「2.5D axial MIL baseline」を、まず4領域mask抜きで構築する設計を確定。
実装は次セッションで Codex に委譲して着手する。

確定設計: `.claude/docs/fracture-baseline-design.md`
参考手法メモ: `.claude/docs/research/dmil-slice-aggregation.md`
Codex検証: `.claude/docs/codex/20260610-2330-fracture-baseline-design.md`,
            `.claude/docs/codex/20260611-loss-aug-schedule-validation.md`

### 方針

- RSNA2022 の CNN+RNN/LSTM 2-stage は 51症例では過学習リスク大 → **end-to-end 2.5D MIL** を採用。
- slice 損失集約は参考PDF (Hibi et al. 2023 IJCARS WSAD) の **DMIL + center loss** を採用。
- メインタスクは **椎体単位の骨折分類**（slice スコアは局在化の副産物）。

### データ仕様（確定）

- 入力: `dataset_zprop/{sample}/{Cx}/images/slice_XXX.png` (224x224 uint8)、`masks/` 全bag完備。
- 椎体ラベル: `add_fracture_labels.py` の `FRACTURE_VERTEBRAE`。
- slice ラベル(評価のみ): `fracture.nii.gz`、`slice_XXX`番号=z-index で整合。
- 規模: **41症例 / 284 bag / 陽性62・陰性222 (21.8%) / slice 44-138 (中央値71)**、pos_weight≈3.58。
- 10症例 (sample6, sample9 等) は CT 未生成 → 後で追加候補。

### 設計の要点（Codex 2回検証で確定）

- モデル: ResNet34 pretrained, 2.5D 3ch `[z-1,z,z+1]`, mask外抑制, per-slice logit, dropout 0.2。
- top-k: **config化**。default capped `k_i=clip(round(0.10*t_i),3,8)`、ratio `ceil(t_i/alpha)` 切替可
  （α を後から調整可能にする方針）。
- DMIL: top-k の **logit** に BCEWithLogits（sigmoid二重適用厳禁）。
- center loss: 陰性bagのみ、自己 detached mean への分散縮小。**β=5**（最初3ep 0→5 warmup, sweep {2,5,10}）。
- 不均衡: **WeightedRandomSampler(~50:50) + 両クラス同一拡張**。陽性だけ強い拡張は禁止（リーク）。
  balanced sampler 使用時、loss側に追加 class weight は重ねない。
- Augment: Affine のみ p=0.7 (rot±7°/trans±4%/scale0.95-1.05)、**flip無し**、intensity は stack/bag 同一。
- LR: **cosine 不採用**。constant + 2ep warmup、**differential LR (backbone 5e-5 / head 2e-4)**、
  AdamW wd1e-4, grad clip1.0, max35ep, early stop val AUPRC patience8。
- CV: StratifiedGroupKFold(5) by patient（リーク厳禁＝最大リスク）。
- 評価: OOF pooled 椎体 AUROC/AUPRC、患者クラスタ bootstrap CI、レベル別 C1/C2/C3-7 pooled (n_pos併記)。

### 次セッションのタスク（→ 実装済みに変更）

配置を `Unet/fracture_baseline/` から **`learning/`** に変更して実装済み（後述）。

### 未確定・後回し

- top-k default は capped、α は sweep 予定（capped vs ratio 比較）。β sweep {2,5,10}。
- PNG の window が骨折向け（bone window）に最適か未確認 → 必要なら HU 再描画を検討。
- slice-AUC 評価の z-alignment 実装（slice_XXX=z-index 前提、要動作確認）。
- 未生成10症例の dataset_zprop 追加（51症例化）。

## 3. 椎体骨折分類ベースライン実装（learning/ 配下）

前セクションの設計を `learning/` に実装した（36テスト全通過）。

### ファイル構成

```
learning/
├── train.py               # エントリポイント（5-fold CV・実効config保存・OOF評価）
├── config/config.yaml     # 全ハイパーパラメータ（augmentation セクション含む）
├── src/
│   ├── model.py           # ResNet34（ImageNet pretrained、dropout+1出力FC）
│   ├── dataset.py         # FractureDataset（2.5Dスタック・mask抑制・RAMキャッシュ）
│   ├── data_utils.py      # load_config / save_effective_config / split_bags_cv
│   └── trainer.py         # 1-fold学習（grad accum 4・AMP・differential LR・early stop）
├── utils/
│   ├── losses.py          # DMIL（top-k BCEWithLogits）+ center loss
│   ├── metrics.py         # AUROC/AUPRC + P/R/F1（threshold=0.5 & F1最適）
│   ├── sampler.py         # WeightedRandomSampler（bag ~50:50）
│   └── augment.py         # Affine + intensity（両クラス共通・config化）
└── test/                  # 36テスト全通過
```

### 追加実装した機能（設計外）

- **実効 config を毎回保存**（`outputs/{run_name}/config.yaml`）
- **P/R/F1 を評価に追加**（threshold=0.5 固定 + F1最適しきい値の両方）
- **epoch ログ**（`fold{N}/training.log`）: train_loss（dmil/center内訳）・val_loss・val_AUPRC・val_AUROC・LR・経過時間・best マーカー
- **RAM プリロード**（`prefetch_to_ram: true`）: NFS IO ボトルネック解消
- **augmentation 全パラメータを config 化**
- `TMPDIR=/tmp` 設定（NFS multiprocessing 一時ディレクトリエラー抑制）

### 学習コマンド

```bash
uv run python -m learning.train 2>&1 | tee learning/outputs/run1/console.log
```
