### 5. ヒートマップ閾値を適応閾値へ変更
- 背景: 固定閾値 `0.5` は低ピークだが線らしい予測を消しやすく、固定低閾値は飛び散りノイズを拾いやすい。
- 方針: 評価・可視化時のモーメント抽出で `threshold = max(min, peak_ratio * peak)` を使う適応閾値を導入。
- 既定値: `min=0.15`, `peak_ratio=0.4`。`peak_ratio` は各ヒートマップ最大値の何割以上を残すかを表す。
- 実装: `utils/losses.py` と `utils/detection.py` の両方に同じ閾値仕様を追加し、`evaluation.py` / `inference.py` / `trainer.py` は dict/string の閾値指定をそのまま流すよう変更。
- 注意: 適応閾値は欠けとスケール差への一次対策であり、飛び散りが強い場合は次段で最大連結成分フィルタや RANSAC/TLS の追加が必要。
- 検証: `uv run pytest Unet/line_only/test/test_line_losses.py -q` は **22/22 pass**、対象ファイルの `ruff check` も pass。
