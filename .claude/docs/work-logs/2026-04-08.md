## やったこと

### multitask アーキテクチャ仕様書の策定

`Unet/memo/multitask_architecture_spec.md` を作成。

**方針決定:**
- `Unet/multitask/` を `line_only/` と完全独立に新設（共通コードはコピー）
- `line_only/` には一切手を入れない

**アーキテクチャ:**
- ResUNet: shared encoder + dual decoder（seg / line 完全独立）
- チャネル幅: [24, 48, 96, 192]（Codex推奨: ~1.58M params、500スライスに適切）
- Residual block: Pre-activation 方式（GN→ReLU→Conv→GN→ReLU→Conv + shortcut）
- GroupNorm: 8 groups 固定、Conv bias=False
- Downsampling: MaxPool2d(2)、Upsampling: ConvTranspose2d

**損失（確定）:**
- `L = L_line + α·L_seg`、α=0.03
- 部分教師: ラベルありサンプルのみで CE mean（分母から除外）
- バッチ構成: 3〜4/8 を seg GT ありに調整（WeightedRandomSampler）

**Codex レビューで修正した点:**
- チャネル幅を [32,64,128,256]（2.81M）→ [24,48,96,192]（1.58M）に縮小
- ResBlock を Post-activation → Pre-activation に変更
- α を 0.1 → 0.03 に修正（MSE vs CE のスケール差を考慮）
- 部分教師の集計方法を「ゼロ埋め全体平均」→「ラベルありのみ mean」に修正

### 線損失設計の調査・議論

**Gemini 調査結果** (`.claude/docs/research/line-loss-design-research.md`):
- Adaptive Wing Loss, WAHR, Boundary DoU など最新手法を確認
- Multitask バランスには Uncertainty Weighting / GradNorm が有効

**設計決定（確定）:**

| 項目 | 決定 |
|------|------|
| ヒートマップ損失 | **MSE のみ**（Adaptive Wing Loss 等は将来候補） |
| 幾何損失（angle/rho） | **導入しない**（初期実験では） |
| 損失式 | `L = L_line(MSE) + 0.03·L_seg(CE)` で確定 |
| 式の主従 | line 主・seg 補助（GT マスクが線から生成されているため線が基盤） |
| Sampler | seg-labeled を 3〜4/8 確保（WeightedRandomSampler） |
| 動的重み | 初期は不採用、negative transfer 観測時に検討 |
