### アルゴリズム概要

1. 各ポリライン → PCAフィット → `FittedLine`
2. `j_r` = line_1/line_2 最近傍端点ペアの中点、`j_l` = line_3/line_4 同様
3. 各直線のbody側法線を決定:
   - line_1: j_l方向、line_3: j_r方向、line_2/line_4: 上方向（anterior）
4. 各椎骨ピクセルを符号付き距離で分類:
   - body = s1+ ∩ s3+ ∩ s2+ ∩ s4+
   - right_foramen = s1- ∩ s2+（rightが優先）
   - left_foramen = s3- ∩ s4+（rightと重複しない分）
   - posterior = 残り
5. 5ch one-hot マスク構築

## TotalSegmentatorマスク品質問題

### 問題

TotalSegmentatorが生成する椎骨マスク（outer boundary）が歪んでいる場合、  
領域マスクの品質も低下する。inner boundary（ライン注釈）は正確だが、  
outer maskへの依存が品質ボトルネックになっている。

### Codex推奨対策（優先順）

1. **QCフィルタリング**（工数低・即効）
   - 指標: `solidity`、隣接スライス連続性、line端点がmask内か、領域面積比率
   - 結果: `exclude / downweight / keep` に分類

2. **eroded maskベース化**（中期）
   - 椎骨マスクを少し収縮してから分割 → 外周歪みの影響を遮断
   - half-plane partitionのline主体設計はそのまま維持

3. Training時: QCスコアによるcurriculum learning（補助）

詳細: `codex/20260406-mask-quality-strategy.md`

## convert_to_png.py: CT-mask affineズレ修正・dataset再生成

### 問題

SlicerでCTをcropして保存したが、maskは保存し直していなかったため、  
`ct_nii.affine` と `mask_nii.affine` が最大 **25.6mm** ずれていた。  
修正前は同じ `slice_idx` でそのまま切っていたため、PNGのCTとmaskが空間的にずれていた。

### 修正内容（L309-311）

```python
if not np.allclose(ct_nii.affine, mask_nii.affine) or ct_nii.shape != mask_nii.shape:
    print('  警告: CTとマスクのボクセルグリッドが異なります。マスクをCT空間にリサンプリングします。')
    mask_nii = nibabel.processing.resample_from_to(mask_nii, ct_nii, order=0)
```

- `order=0`（最近傍補間）でバイナリ値を保持
- 全41サンプルのうち、ほぼ全件で警告が発動 → 問題が広範に存在していた

### TotalSegmentatorマスク品質問題との関係

- TotalSegmentatorマスクの歪み問題（上記セクション参照）と今回のaffineズレは**別問題**
- ただし、**両方が重なってmaskの品質を下げていた**可能性がある
  - affineズレ → CTとmaskが空間的に一致しない
  - TotalSegmentator品質 → maskの形状自体が歪む
- 今回の修正でaffineズレは解消。残る問題はTotalSegmentatorの品質のみ
