## 作業内容①: 椎体axial断面の自動tilt補正スクリプト作成

### 背景
`predata_simple/` にある椎体データのアノテーション作業で、axial面に椎体が正しく映るよう
手動で断面を傾ける作業が重労働だった。これを自動化したい。

### アプローチ
- 「axial断面にした時、椎体マスクが途切れず最大面積になる向き＝最適tilt」という考えに基づく
- coarse-to-fine の2段階グリッドサーチで最適な (rx, ry) 角を探索
  1. 0.5倍ダウンサンプルで ±30°, step5° のcoarse search
  2. 元解像度で best付近 ±5°, step1° のfine search

### 実装
`Unet/preprocessing/auto_tilt.py` を新規作成

- `find_best_tilt(mask)` : coarse→fine探索で最適角度を返す
- `apply_rotation(data, rx, ry)` : scipy affine_transform で3Dボリューム回転
- アノテーション済み（`.mrk.json` あり）の椎体は自動スキップ
- 出力: `annotation_data/sampleX/CY/{ct.nii.gz, mask.nii.gz, rotation_deg.npy}`

### 実行結果
- アノテーション済み 139件: スキップ
- 未処理 148件: tilt補正して `annotation_data/` に保存
- エラー: 0件

## 作業内容②: ラインアノテーションツール + スライス抽出スクリプト作成

### 背景
未アノテーション椎体（152件）に lines.json を付けるため、
VSCode上でラインを引けるブラウザツールを作成。

### 実装したファイル

#### `Unet/preprocessing/extract_slices.py`（新規）
`predata_simple/` から直接、tilt補正済みスライスPNGを `dataset/` に抽出する。

**方式: 斜断面サンプリング（oblique slicing）**
- 3Dボリュームを回転して保存するのではなく、傾いた平面を直接サンプリング
- `scipy.ndimage.map_coordinates` で各出力ピクセルを元ボリュームから1回サンプリング
- 回転補間によるぼけが出ない
- 回転後z軸を .T（転置）して表示軸を convert_to_png.py と揃える

**スライス選択: ピーク前後 ±5 スライス**
- マスク面積が最大になるスライス位置を検出
- その前後5スライス（計11枚）から n_slices 枚を等間隔選択
- 端の薄いスライスを除外

```bash
# 全サンプル処理（lines.json 未作成のものだけ）
uv run python Unet/preprocessing/extract_slices.py

# 1サンプルのみ
uv run python Unet/preprocessing/extract_slices.py --sample sample10 --vertebra C4

# 上書き（lines.json 済みも再生成）
uv run python Unet/preprocessing/extract_slices.py --sample sample28 --vertebra C1 --overwrite
```

```bash
uv run python Unet/annotation_tool/server.py
```

#### `Unet/annotation_tool/index.html`（新規）
Canvas ベースのアノテーションUI。

- 左サイドバー: サンプル一覧（アノテーション済み/未済バッジ表示）
- 画像+マスクオーバーレイ表示
- line_1〜4 をクリックで折れ線描画
- キー操作: 1〜4（ライン選択）、Ctrl+Z（Undo）、Ctrl+S（Save）、←→（スライス移動）
- 保存先: `dataset/sampleX/CY/lines.json`

### 判明した問題と解決

| 問題 | 原因 | 解決 |
|------|------|------|
| dataset/sample36/C2 のスライスがおかしい | auto_tilt済みNIfTIを保存→再読み込みすると品質劣化 | predata_simple から直接サンプリング |
