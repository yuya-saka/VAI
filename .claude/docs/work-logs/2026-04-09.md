## 現在の問題意識

**multitask の heatmap が line_only より"太く・鈍く"なる negative transfer が起きている**

- perp はほぼ同じだが peak/angle/rho が悪化
- 原因: dense な seg 勾配が sparse な line 信号を圧倒している
- epoch 28 以降: seg_miou は改善し続けるが line metrics は振動・停滞

## 次セッションの議論

**seg_loss の影響を抑えながら heatmap 品質を改善する方法:**

1. `alpha_seg` を下げる（現在 0.03）
   - seg head は seg_loss なしでは学習できない（line_loss の勾配は seg head に流れない）
   - いくつが適切か要検討

2. Codex 推奨の追加対策:
   - seg loss の遅延導入（最初 10〜20 epoch は alpha_seg=0）
   - line_only checkpoint で encoder 初期化 → seg head だけ立ち上げ → unfreeze
   - weighted MSE heatmap loss（peak 帯を強調）

3. C1/C7 の vertebra-aware weighting（line loss のみに C1: 1.75〜2.0x, C7: 1.25〜1.5x）
