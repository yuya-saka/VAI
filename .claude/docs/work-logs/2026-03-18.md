# 2026-03-18: Critical Bugs Fixed & Y-Axis Coordinate Bug Discovery

## 重大バグ修正（Codex分析）

`line_losses.py` と `train_heat.py` の3つの重大バグを修正：

**1. NaN処理の順序（line_losses.py:191-278）**
- 問題: NaNを含むテンソルに `cos/sin/exp` を計算してからマスク処理
- 影響: `NaN * 0 = NaN` がバッチ損失を汚染、学習崩壊
- 修正: 先に有効エントリを抽出し、有効データのみで損失計算
- 追加: confidence に `.detach()` を追加して勾配崩壊防止

**2. ρ損失のDouble Smoothing（line_losses.py:265）**
- 問題: smooth minimumを適用後、さらに `F.smooth_l1_loss()` を適用
- 影響: 数学的に誤り、勾配スケールが不安定
- 修正: `F.smooth_l1_loss()` 呼び出しを削除

**3. Warmup重み式（train_heat.py:843）**
- 問題: `loss = loss_mse + warmup_weight * line_loss`（MSEは常に定数）
- 影響: epoch 50でジオメトリ損失が有効化される際に勾配ショック
- 修正: `loss = (1 - 0.5*warmup_weight) * loss_mse + warmup_weight * line_loss`

**修正後の結果:**

## Y軸座標系バグの発見

**根本原因（line_losses.py:96）:**
```python
# 誤（画像座標系）:
y_grid = torch.arange(H, ...) - H / 2.0
# 行0（上）→ -H/2、行H-1（下）→ +H/2（Y下向き増加）

# 正（数学座標系）:
y_grid = -(torch.arange(H, ...) - H / 2.0)
# 行0（上）→ +H/2、行H-1（下）→ -H/2（Y上向き増加）
```

**なぜ40-50°の誤差が出るか（90°や180°でなく）:**
- 反転したY軸によりモーメント計算で非線形な角度歪みが発生
- 20°の直線: mu11の符号が非対称に反転 → ~43°の誤差
- 125°の直線: 同様のメカニズム → ~52°の誤差

**ユニットテストで修正効果を確認:**
- sample5_C1_slice029 line_1: 42.72° → **0.03°**
- sample5_C1_slice029 line_3: 52.64° → **0.02°**
- 全テスト角度（0°-180°）: 最大誤差 **0.38°**

**再訓練後の期待値:**
- angle_error: 34.30° → **<10°**
- rho_error: 15.75px → **<10px**
