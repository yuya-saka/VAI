## 3. 外れ値分析: モーメント法の失敗モード特定

`sig4.0_ALL(brob追加)` の error_viz を実行し worst 例を分析。

**問題**: `sample54_C7_slice065` line_3 で 64.5° の角度誤差。  
**原因**: モーメントの距離² 重み付けにより、メインブロブから 146px 離れた小さな過検出が

## 4. CC（連結成分）フィルタを `detect_line_moments` に実装

`utils/detection.py`:
```python
binary = (hm > 0).astype(np.uint8)
labeled, n_components = ndimage.label(binary)
if n_components > 1:
    peak_y, peak_x = np.unravel_index(hm.argmax(), hm.shape)
    main_label = labeled[peak_y, peak_x]
    hm = hm * (labeled == main_label)
```
ピーク値を含む連結成分のみ残し、離れた過検出ブロブを除去。

実際の効果確認（fold1 チェックポイントに直接適用）: sample54_C7_slice065 line_3 が 64.5° → 6.8°。

## 8. 領域分割評価パイプライン構築（2026-06-17）

### 概要

`sig4.0_ALL(CC適用)` の予測線から領域分割を生成し、`data/dataset_zprop` GT と 3D Dice で評価。

### 評価手法

1. 予測スライス → `generate_region_mask` で 4 領域マスク生成
2. 非外れ値スライス（angle_error < 10°）をアンカーとして `propagate_lines_z` と同手法で z 伝播
3. 外れ値スライスは内挿、非テストスライスは外挿
4. 評価指標: **3D (volumetric) Dice**（椎骨ごとにピクセル積算）← per-slice Dice より適切

### 主な知見

| 指標 | 全由来 | 外挿のみ |
|------|:------:|:-------:|
| 3D Dice mean（284 椎骨） | **0.829** | 0.798 |
| class1 椎体 | 0.898 | 0.887 |
| class2 右椎間孔 | 0.740 | 0.683 |
| class3 左椎間孔 | 0.749 | 0.700 |
| class4 後方要素 | 0.929 | 0.920 |

- per-slice Dice (0.750) より 3D Dice (0.829) が大幅高い → 端スライスの GT 極小問題が原因
- 外挿スライスの GT 面積は anchor の 1/3 以下（median 2,955px vs 7,100px）
- 椎間孔(c2/c3)の低精度は「細いくさび形領域への junction ズレの影響」+「GT 自体が小さい」両方
- anchor 並み面積(≥5000px)に絞ると外挿 Dice 0.808（椎体・後方は 0.9 超）
