## 3. 入力と座標系

### 入力


### DICOM seriesの構築

- `ImageOrientationPatient`からrow方向・column方向を取得する

### 座標規約

- 基準座標はDICOM patient coordinateのLPS mmとする
- TotalSegmentatorのNIfTI maskを扱う場合は、NIfTI affineを通して物理座標へ変換する
- NIfTI voxel indexを症例間で共通の基準として使用しない
- すべてのcrop、回転中心、平面位置、bbox変換は物理座標で記録する

## 4. TotalSegmentator

ローカル環境のTotalSegmentatorは、入力としてNIfTIだけでなくDICOM folderまたはDICOM zipを受け付ける。

### 基本方針

```text
TotalSegmentator
  - input: DICOM series directory
  - roi_subset: vertebrae_C1 ... vertebrae_C7
  - output: C1-C7 maskのみ
```

- 第一選択はDICOM directoryの直接入力
- DICOM直接入力で失敗する症例のみ、一時NIfTIへfallbackする
- 一時NIfTIは`/tmp`またはローカルscratchへ症例単位で作成する
- fallback NIfTIはC1-C7処理完了後に削除する
- CT全体のNIfTIを`data/rsna_data/nifti/`へ全件保存しない

### 永続保持するsegmentation

```text
data/rsna_data/segmentations/{StudyInstanceUID}/
├── vertebrae_C1.nii.gz
├── ...
└── vertebrae_C7.nii.gz
```

C1-C7 maskは再現性・QC・再処理のため永続保持する。CT NIfTIと椎骨crop NIfTIは保持しない。

## 5. Mask cleaningと物理ROI

各椎骨maskに以下を適用する。

1. binary化
2. 3D connected component labeling
3. 最大連結成分のみ保持
4. voxelをmask affine経由でpatient coordinateへ変換
5. 物理重心、物理bbox、体積、頭尾方向投影範囲を計算

QCでは以下を確認する。

- C1-C7の期待maskが存在する
- maskが空でない
- 異常に小さい・大きいcomponentでない
- 隣接椎骨との位置関係が大きく逆転していない
- CTとmaskの物理空間が一致する

ROIはvoxel数ではなくmmで表現する。最終2D平面の中心は原則として椎骨maskの物理重心とする。

## 6. 等方性サンプリングを行わない範囲

行わない処理:

- CT全体を`1×1×1 mm`へ変換
- CT全体を`0.4×0.4×0.4 mm`へ変換
- 椎骨cropを等方性3D NIfTIとして保存
- z方向に補間した全volumeを永続保存

行う処理:

- 必要な2D傾斜平面だけを元CTから直接サンプリング
- 各出力平面を既存線検出モデルと同じ`224×224`へラスタライズ

最終平面は旧データセットとの整合を優先し、初期値として以下を使用する。

```text
in-plane resolution: 0.4 mm/pixel
image size:          224 × 224
physical FOV:        89.6 × 89.6 mm
```

これは2D出力平面内のサンプリングであり、CT全体の3D等方化ではない。CTはlinear interpolation、maskとlabel mapはnearest-neighbor interpolationを使用する。

## 8. 骨折分類用15平面

分類用15平面は、椎骨中央だけでなく上端から下端までを覆う。

### 有効範囲

1. cleaned maskの各voxelを補正後の頭尾方向軸へ射影する
2. mask体積の累積分布を作る
3. 累積`1%`位置を`t_low`
4. 累積`99%`位置を`t_high`

孤立voxelや薄いmask tipの影響を抑えつつ、椎骨全体を維持する。

### 15平面の位置

```python
slice_positions_mm = np.linspace(t_low, t_high, 15)
```

- bboxアノテーションがある学習症例では、連続するbboxスライス区間ごとに実bboxスライスを1枚選び、その物理位置を15平面へ必須挿入する
- bbox必須位置が`t_low`から`t_high`の外側にある場合は、bbox位置を含むよう有効範囲を拡張する
- bbox必須位置を固定した後、残り枠を椎骨全長のカバレッジが最大になるよう配置する
- bbox由来平面は`bbox_forced`としてmetadataへ記録する
- DICOMのslice indexやslice thicknessには依存しない
- 各位置で同じ補正後平面基底を使う
- 各平面を元CTから`224×224`へ直接サンプリングする
- 旧アノテーション用の「最大面積位置±5スライス」は使用しない

0番目が頭側、7番目が中央付近、14番目が尾側となるよう順序を固定する。

## 9. 線予測用アンカー平面

分類用15平面の端部では、4本の解剖学的境界が不明瞭になる可能性がある。そのため、分類用15平面すべてへ線検出モデルを直接適用しない。

### 候補平面の生成

- 補正後の頭尾方向軸に沿い、約`1 mm`間隔で候補平面を一時生成する
- 候補範囲はcleaned maskの連続した有効範囲とする
- 候補CT・maskは処理中のみ保持し、永続保存しない

### 事前除外

以下の候補は線モデルへ入力しない、またはアンカー対象外とする。

- mask断面積が極端に小さい
- maskのsolidityが著しく低い
- 小さい孤立componentが断面形状を支配している
- `224×224` FOVから主要構造が切れている

### 線検出後の品質評価

未知症例ではGT角度誤差を使えないため、以下の推論可能な指標を使用する。

- 4 heatmapすべてのpeak
- 主連結成分のmass比
- moment anisotropy / line confidence
- 4本すべてのendpointsが有効
- `J_R.x > J_L.x`
- junction間距離が退化していない
- 生成した4領域がトポロジー条件を満たす
- 隣接候補との角度・位置変化が連続している

トポロジー不正はhard rejectとする。その他の閾値とscore重みは小規模pilotで決定する。

### アンカー選択

1. 有効な候補範囲を頭尾方向に5 binへ分割する
2. 各binから最高品質の候補を1枚選ぶ
3. 重複候補は除く
4. 通常5枚、最低3枚をアンカーとする

全体上位5枚だけを選ぶと最大面積付近へ集中するため、位置binによって頭尾方向のカバレッジを確保する。

3アンカー未満の場合は自動的に低品質な予測を採用せず、vertebra単位のQC failureとして記録する。

## 10. 4線のz方向伝播

以下の既存実装を再利用する。

- `data_preprocessing/learning_dataset/propagate_lines_z.py`
- `data_preprocessing/segmentation_dataset/generate_region_mask.py`

### 伝播状態

- 右junction軌跡`J_R(t)`
- 左junction軌跡`J_L(t)`
- 4本の線方向
- 線方向は180°周期を扱うためdoubled-angle表現を使用
- 座標は各平面のmask centroidとbbox scaleで正規化する

### 伝播方法

- アンカー間: 平滑な内挿
- アンカー外: 変位・回転量を制限した外挿
- 新実装では離散z-indexではなく物理位置mmを独立変数とする
- 15分類平面それぞれで線を再構築する
- 各平面のvertebra maskと4線から4領域label mapを生成する

### provenance

各15平面へ以下を付与する。

- `anchor`
- `interpolated`
- `extrapolated`
- 最近傍アンカーまでの距離mm
- line confidence
- propagation confidence
- topology QC

低confidenceの領域maskは削除せずmetadataで明示し、学習loader側でdownweightまたは無効化できるようにする。

## 13. Bounding boxラベルの整合

対象は`train_bounding_boxes.csv`の235症例。

元bboxはDICOM axial画像上の`x, y, width, height, slice_number`で定義されている。NIfTI indexを中間表現として使用せず、以下で変換する。

1. `slice_number`に対応するDICOM SOP Instanceを特定する
2. bboxのpixel座標を`ImagePositionPatient`、`ImageOrientationPatient`、`PixelSpacing`でpatient coordinateへ変換する
3. bbox位置の物理平面で、伝播済み4線と4領域を評価する
4. bbox polygonと4領域のoverlapを計算する
5. 最大overlap領域へ骨折region labelを付与する

bboxを最近傍の15分類平面へ単純丸めしない。連続するbboxスライス区間ごとに実bboxスライスを1枚選び、その中心の物理位置を15分類平面へ必須挿入する。bboxの領域判定自体は元物理位置で行い、選択された平面へ`bbox_forced` provenanceを保存する。

出力:

```text
region_labels.csv
StudyInstanceUID, vertebra, region, fracture, source_slice, qc_status
```

## 21. TotalSegmentator runner実装

### 実装

- `data_preprocessing/rsna_pipeline/totalsegmentator_runner.py`を追加
- DICOM directoryをTotalSegmentatorへ直接渡す
- direct入力失敗時のみ、`dcm2niix`で症例単位の一時NIfTIを作成して1回retry
- 一時NIfTIとworker結果は既定で`/tmp/rsna_totalsegmentator`へ保存し、処理後に削除
- C1-C7の7 maskすべてが存在し、non-emptyである場合のみ完了扱い
- final directoryへ直接書かず、staging directoryで検証後にatomic replace
- 既存の部分出力は、完全な新規出力が生成できた場合のみ置換

### 負荷制御

- default workers/GPU: `1`
- default global CPU thread budget: `8`
- mask saving threads: workerあたり最大`2`
- `OMP_NUM_THREADS`、`MKL_NUM_THREADS`、`OPENBLAS_NUM_THREADS`、`NUMEXPR_NUM_THREADS`をworker単位で制限
- TotalSegmentatorはworker内で遅延importし、親processでCUDAを初期化しない
- 高並列化は`--workers-per-gpu`、`--max-workers`、`--cpu-thread-budget`で明示指定した場合のみ行う

- direct DICOM成功、temporary NIfTI fallback、部分mask拒否、worker layoutの単体テストを追加

- 実症例`1.2.826.0.1.3680043.10041`のfallback変換:
  - NIfTI shape: `(512, 512, 220)`
  - compressed size: 約`51.7 MB`
  - conversion time: 約`1.52`秒
- 現実行環境ではNVIDIA driverへ接続できず、GPU segmentation pilotは未実施
