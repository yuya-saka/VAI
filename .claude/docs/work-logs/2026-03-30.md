### 2. fold1 回帰調査（完了）

sig2.0_base（新 src コード）で fold1 角度誤差が **8.60° → 17.77°** に悪化した原因を調査。

**実験: ShiftScaleRotate vs Affine の影響測定**

`test/test_aug_fold1.py` と `test/test_aug_fold1_修正.py` で比較：

| 実験 | val angle |
|------|-----------|
| ShiftScaleRotate | 5.39° |
| Affine | 5.19° |

→ **augmentation の変更は原因ではない**（差 0.20°）

**注意:** 旧 `test_aug_fold1.py` にパッチバグあり。`data_utils.py` は `from .dataset import get_transforms` で既にバインドされているため、`dataset_module.get_transforms` だけパッチしても効かない。修正版は両モジュールをパッチ。

**sig2.0_base_debug 実験（全 fold）:**

| fold | debug | 精度悪化したやつ | baseline |
|------|-------|----------------|----------|
| fold0 | 5.57° | 5.94° | 5.43° |
| fold1 | **7.60°** | **17.77°** | 8.60° |
| fold2 | 5.08° | 5.14° | 4.94° |
| fold3 | 4.26° | 5.11° | 4.78° |
| fold4 | 7.99° | 8.11° | 8.11° |
| **AVG** | **6.10°** | **8.41°** | **6.37°** |

→ debug は baseline より良い平均精度（6.10° vs 6.37°）
→ fold1 の一時的劣化は debug では解消（17.77° → 7.60°）

**Codex 根本原因分析（`.claude/docs/codex/20260330-codex-fold1-direct.md`）:**
- fold1 は最初の「finish→reinit 境界」を跨ぐ fold → 微小な非決定性が最も増幅されやすい
- **推奨修正**: `--start_fold 1 --end_fold 1` でfoldごとに別プロセス実行し reinit=True 連鎖を止める
- wandb=true で実験し直したところ精度異常は再現せず → **非決定的挙動と確定**

**結論: 新 src コードは正常、fold1 劣化はコードの問題ではない**

### 3. 椎体ラベル条件付き U-Net の実装（完了）

`Unet/memo/椎体ラベル条件付き U-Net 実装計画メモ.md` の方針に沿って実装。

**背景:** C1〜C7 は解剖学的形状が大きく異なるため、椎体ラベルを条件として与えることで学習の安定化を狙う。

**実装方針（Codex 確認済み）:**
- bottleneck concat + one-hot が小規模データ（<100患者）には最適
- FiLM conditioning が次点（次の実験ステップ候補）
- multi-level injection は過学習リスク最大で非推奨

**実装内容:**

| ファイル | 変更内容 |
|---------|---------|
| `src/model.py` | `VERTEBRA_TO_IDX`、`num_vertebra` パラメータ、`cond_proj`（identity初期化）、`_onehot_map`、`forward(x, vertebra_idx=None)` |
| `src/data_utils.py` | `use_vertebra_conditioning` フラグ読み込み |
| `src/trainer.py` | 4箇所の `model(x)` → `model(x, v_idx)` に変更 |
| `config/config.yaml` | `use_vertebra_conditioning: true` + `num_vertebra: 7` を追加 |

**identity初期化:** `cond_proj` の weight を identity 初期化することで、`t=0` で旧モデルと全く同じ出力になる。既存 checkpoint からの継続訓練も安全。

**テスト結果:** 33 passed
