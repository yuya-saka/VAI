### 実施内容

### 実装した変更

#### 1. `mp.spawn` による自動マルチGPU起動

**背景:** 以前は `torchrun --nproc_per_node=2` コマンドが必要だったが、`uv run python train.py` 一本で 1GPU/2GPU 両方動くようにしたい。

**対応:** `train.py` を `torch.multiprocessing.spawn` ベースに全面書き直し。

```
main()            ← config を読んで n_gpu を確認、プロセス起動
_worker()         ← DDP初期化 (dist.init_process_group) → _do_training → destroy
_do_training()    ← 実際の学習ロジック (is_main で rank-0 のみ出力)
_find_free_port() ← NCCL 通信用ポートを自動選択
```

- `CUDA_VISIBLE_DEVICES` は `main()` 内で CUDA 初期化前に設定（`gpu_ids: [0,2]` から自動生成）
- `n_gpu: 1` のときは `mp.spawn` 不使用、直接 `_do_training(0, 1, ...)` を呼ぶ

#### 2. batch_size・LR の調整経緯

Linear Scaling Rule: `新LR = base_lr × (実効バッチ / 基準バッチ8)`

| 実効バッチ | n_gpu | batch_size/GPU | LR |
|-----------|-------|---------------|----|
| 8 | 1 | 8 | 2.3e-4 (base) |
| 16 | 2 | **8** | 4.6e-4 |
| 32 | 2 | 16 | 9.2e-4 |

現在の config (`v1_gpu2`) は実効16（per-GPU=8, n_gpu=2, LR=4.6e-4）で設定済み。

