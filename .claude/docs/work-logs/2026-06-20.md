## Physical orientation correction

### 実装

- `data_preprocessing/rsna_pipeline/orientation.py`を追加
- cleaned maskの物理ROIのみをmemory上の`0.4 mm`等方gridへresample
- grid軸はDICOM row / column / normal basisに整合
- 既存`auto_tilt.py`と同じ最大axial mask面積を目的関数として使用
- coarse search: `±30°`, `5°` step
- fine search: coarse最良値の周囲`±5°`, `1°` step
- 勝者角度から補正後のLPS row / column / normal basisを生成
- `±30°`境界への張り付きをQCへ記録
- CT全体・mask全体の等方化volumeは作成・保存しない

### 検討中に棄却した方式

- 異方性maskへ旧voxel-index回転を直接適用
  - 物理角度がspacingに依存し、full volume回転が非常に遅い
- mask voxel中心のprojection histogram
  - slice spacingによる軸整列biasが発生
- voxel投影厚を用いた断面密度近似
  - 微小傾斜の断面積を過大評価

### 実データ確認

- 症例`1.2.826.0.1.3680043.10041`
- C1-C7統合処理時間: 約`67.6秒`
- orientation search boundary: `0 / 7`
- 結果:
  - C1: `(rx, ry) = (17°, -4°)`
  - C2: `(20°, -14°)`
  - C3: `(12°, -9°)`
  - C4: `(12°, -7°)`
  - C5: `(11°, 0°)`
  - C6: `(11°, -10°)`
  - C7: `(0°, -9°)`

### 検証

- axis-aligned ellipsoid: `0°`を復元
- known `20°` tilted ellipsoid: 傾斜を復元
- 補正後basisの直交性・右手系を確認

### 次の作業

1. 補正後normalへcleaned mask voxelを射影
2. 累積mask体積の`1%-99%`位置を計算
3. bbox必須位置を含む15分類平面を決定
4. 15平面のorigin / basis / normalをmetadataへ追加
