### Y軸符号バグの詳細

`compute_perpendicular_distance` の修正内容：

```python
# 現在（バグ）
sampled_points_centered = sampled_points - center  # Y反転なし

# 修正後
x_centered = sampled_points[:, 0] - center
y_centered = -(sampled_points[:, 1] - center)  # Y反転必要
distances = np.abs(x_centered * nx + y_centered * ny - rho_px)
```

### 大きな誤差の原因調査

**仮説：V字型ポリラインが角度誤差を引き起こす**

`Unet/debug/analyze_vshape_correlation.py` でV字度と角度誤差の相関を計算。

| 指標 | 値 |
|------|---|
| Pearson r | -0.012（p=0.49、有意でない） |
| Spearman r | 0.104（弱い） |
| n | 3,356 |
| V字度>5px の割合 | 97.4% |

**結論：V字型は角度誤差の主要因ではない。**

理由：モデルがV字型ヒートマップを学習しており、GT抽出（PCA）と予測抽出（PCA）が同じ方向を返すため誤差が小さくなる。

大きな誤差の残る原因候補：
- GTアノテーション品質（bad_slices未除外の粗いデータ）
- モデルが等方的ヒートマップを予測（方向不安定）
- 予測が複数モードに分散
