## データ拡張修正: ポリライン変換後 heatmap 再生成

### 背景

従来の実装：
1. ポリラインから heatmap を生成（拡張前）
2. heatmap を Albumentations に `"image"` として渡す → bilinear 補間でにじむ
3. バグ: `RandomBrightnessContrast` / `GaussNoise` が GT heatmap にも適用されていた

### 修正方針（Codex 推奨 Option D）

- ポリライン座標を Albumentations の `keypoints` として渡す
- `KeypointParams(format="xy", remove_invisible=False)` を使用
  - `remove_invisible=False` は必須（画像外点削除でポリライン長が不定になるのを防ぐ）
- `line_1` → 標準 `keypoints` 引数、`line_2/3/4` → `additional_targets`
- 拡張後の変換済み座標から heatmap・gt_line_params を再生成

### 変更ファイル

`Unet/line_only/src/dataset.py`（multitask は未変更）

変更内容：
1. `get_transforms()` - `hm1/hm2/hm3/hm4: "image"` を廃止、keypoint_params 追加、常に `A.ReplayCompose` を返す
2. `PngLineDataset._swap_polylines_on_flip()` メソッド新規追加
3. `__getitem__()` - 拡張前 heatmap 生成を削除、拡張後ポリラインから再生成

テスト結果: 32 passed（既存エラー3件は今回の変更と無関係）
