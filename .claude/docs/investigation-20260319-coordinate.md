# 座標系調査結果 (2026-03-19)

## 調査目的
予測線がGT線と8〜28度ずれている原因として、座標系の不一致を調査

## 結論
**座標系は正しく統一されている。問題は別の箇所にある。**

## 検証結果

### 1. 座標グリッド定義の確認
- `line_losses.py` と `line_detection.py` の座標グリッドは完全に一致
- 左上[0,0]: X=-112, Y=+112 (両方とも同じ)
- 数学座標系(Y上向き、中心原点)で統一されている

### 2. 角度抽出の一貫性テスト
**テストケース: GTポリラインと同じ形状のヒートマップ (sample5_C2_slice036)**
```
line_1: GT=174.60度, extract_pred(同形状HM)=174.78度, 差分=0.18度 ✅
line_2: GT=73.35度,  extract_pred(同形状HM)=73.28度,  差分=0.07度 ✅
line_3: GT=11.75度,  extract_pred(同形状HM)=11.87度,  差分=0.12度 ✅
line_4: GT=105.22度, extract_pred(同形状HM)=105.24度, 差分=0.02度 ✅
```
→ GTと同じ形状なら正確に角度が抽出される

### 3. 実際の予測との比較
```
line_1: GT=174.6度, Pred=148.4度, 差分=26.2度 ❌
line_2: GT=73.3度,  Pred=82.3度,  差分=8.9度  ❌
line_3: GT=11.7度,  Pred=40.1度,  差分=28.4度 ❌
line_4: GT=105.2度, Pred=77.2度,  差分=28.0度 ❌
```
→ モデルの予測ヒートマップ自体の形状がGT線の方向とずれている

## 重要な発見
- `detect_line_moments`(方向角) と `extract_pred_line_params_batch`(法線角) は完全に一致
- 両関数は同じヒートマップから一貫した角度を抽出している
- ヒートマップの可視化では形状は大体正しく見える
