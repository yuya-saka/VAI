# 実験結果: alpha_seg sweep（negative transfer 緩和）

**日付**: 2026-04-09  
**実験ディレクトリ**: `Unet/outputs/multitask_seg下げる/`  
**条件**: alpha_seg = 0.005 / 0.01 / 0.02（fold4のみ）  
**比較基準**: `Unet/outputs/multitask_v1/sig3.5/` の fold4 結果（alpha_seg=0.03）

---

## 背景・問題意識

multitask_v1（alpha_seg=0.03）で「heatmap が line_only より太く・鈍くなる」negative transfer が観測された。

- dense な seg 勾配（全ピクセル）が sparse な line 信号を圧倒していると推定
- epoch 28 以降: seg_miou は改善し続けるが line metrics が振動・停滞
- fold4 では特に angle=9.06° と悪化が顕著

**仮説**: alpha_seg を下げることで seg 勾配の影響を抑制し、line 精度を改善できる。

## 考察

### 1. alpha 下げの効果は明確
angle が全設定で大幅改善（v1 比で C1: 12.19°→8.00°、C2: 14.88°→8.26° など）。
seg 勾配の抑制が negative transfer を緩和できていることが確認できた。

### 2. alpha=0.02 が最もバランスが良い
- 直線: angle・peak でトップ、rho も 3.98px で良好
- seg: fg_miou=0.852・fg_mdice=0.917 で v1 fold4 を上回る唯一の設定
- 直線・seg 両方を同時改善できている

### 3. 0.01 以下は下げすぎ
- alpha=0.01: perp は最良（11.51px）だが rho=4.35px・seg が劣化
- alpha=0.005: perp・angle ともに 0.02 より悪く、最小 alpha が最悪 perp（12.21px）
- 「下げすぎると逆効果」が実験で確認された

### 4. C1・C2 の angle 改善が顕著
v1 fold4 で C1=12.19°, C2=14.88° と突出していた問題が、alpha=0.02 で C1=8.00°, C2=8.26° に改善。
これらの椎骨は seg 勾配の干渉を特に受けやすい可能性がある（形状の特殊性・サンプル分布）。

### 5. fold4 はバリアンスが大きいことに注意
v1 の 5-fold 平均 angle=6.07° に対し fold4 のみ 9.06° と突出して悪かった。
本実験は fold4 のみのため、改善効果の過大評価に注意が必要。

---

## 結論

**alpha=0.02 を暫定ベストとして採用。**

v1 fold4（α=0.03）比:
- angle: 9.06° → 6.48°（-2.58°）
- perp: 12.60 → 11.91px（-0.69px）
- rho: 4.57 → 3.98px（-0.59px）
- fg_miou: 0.849 → 0.852（+0.003）
