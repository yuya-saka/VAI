# 椎骨4領域分割を用いた頸椎骨折局在化研究 草案

## 1. 研究の背景

頸椎骨折は、骨折そのものによる疼痛や不安定性だけでなく、脊髄損傷、神経障害、血管損傷などの重篤な合併症につながる可能性がある。特に、椎骨動脈損傷（Vertebral Artery Injury: VAI）は、受傷直後には無症状であっても、血栓形成や塞栓によって後方循環系脳梗塞を引き起こす可能性があり、早期のリスク評価が重要である。

既存の頸椎骨折検出AIの多くは、患者単位またはC1〜C7の椎体単位で骨折の有無を分類する。しかし、臨床的には「どの椎体が折れているか」だけでなく、「その骨折が横突孔周辺に及んでいるか」「後方要素に存在するか」「椎体中心部に限局しているか」といった解剖学的局在が重要である。特に横突孔周辺の骨折は、椎骨動脈損傷リスクと関連する可能性があり、単なる椎体単位の分類では臨床的なリスク評価として不十分である。

本研究では、頸椎を解剖学的に4領域へ分割し、その領域情報を骨折検出モデルの解釈および局在化に利用することで、椎体単位の骨折分類から領域別骨折リスク提示へ発展させることを目指す。

---

## 2. 現在の到達点

現在までに、頸椎CT画像に対して以下の処理を実装している。

* TotalSegmentatorによるC1〜C7の椎体抽出
* 椎体ごとの軸補正
* axialスライス画像の生成
* 椎体マスクの生成
* 椎体を以下の4領域へ分割する処理

### 4領域の定義

1. 左横突孔領域
2. 右横突孔領域
3. 椎体中心部領域
4. 後方要素領域

また、教師ラベルとしては以下を利用可能である。

* 椎体ごとの骨折ラベル

  * 例：C5 骨折あり / なし
* スライスごとの骨折ラベル

  * ただし、完全に正確であるとは限らない
  * 骨折範囲の目安として利用する

---

## 3. 研究上の問題意識

4領域分割を実装しただけでは、直接的に骨折分類精度が向上するとは限らない。

理由は、現在の主教師ラベルが椎体単位の骨折有無であり、各4領域に対する正解ラベルが存在しないためである。例えば、C5に骨折があることは分かっていても、その骨折が左横突孔領域にあるのか、右横突孔領域にあるのか、椎体中心部なのか、後方要素なのかは教師として与えられていない。

そのため、4領域マスクを単純に入力チャンネルとして追加しても、モデルがどの領域に注目すべきかを明示的に学習できるとは限らない。

したがって、本研究では4領域情報を単なる入力特徴としてではなく、以下の目的で活用する。

1. 骨折分類モデルの出力を解剖学的領域ごとに分解する
2. スライスごとの骨折スコアを4領域ごとに集計する
3. 領域別骨折リスクスコアを提示する
4. 横突孔領域への骨折波及をVAIリスク提示につなげる

---

## 4. 研究目的

本研究の目的は、椎体単位の骨折ラベルと不完全なスライスラベルを用いて、頸椎骨折の領域別局在化を行うことである。

具体的には、2.5D axialスライス入力を用いた椎体単位の骨折分類モデルを構築し、その出力を4領域マスクによって解析することで、左横突孔、右横突孔、椎体中心部、後方要素のどの領域に骨折が疑われるかを提示する。

最終的には、横突孔領域に高い骨折スコアが認められる場合に、VAIリスクが高い可能性を示す診断支援システムへの発展を目指す。

---

## 5. 提案手法の概要

### 5.1 基本方針

本研究では、まず椎体単位の骨折分類ベースラインを構築する。その後、不完全なスライスラベルを補助教師として利用し、スライスごとの骨折スコアを出力できるモデルへ拡張する。

さらに、得られたスライススコアまたは特徴マップを4領域マスクで集計し、領域別骨折スコアを算出する。

### 5.2 入力

1椎体を1つのbagとして扱い、その中のaxialスライスをinstanceとする。

各スライスの入力には、中央スライスだけでなく前後スライスも含める。

例：

```text
Input(z) = [CT(z-2), CT(z-1), CT(z), CT(z+1), CT(z+2)]
```

必要に応じて、椎体マスクを追加する。

```text
Input(z) = [CT(z-2:z+2), vertebra_mask(z)]
```

将来的には4領域マスクも利用する。

```text
Input(z) = [
  CT(z-2:z+2),
  vertebra_mask(z),
  left_TF_mask(z),
  right_TF_mask(z),
  body_mask(z),
  posterior_mask(z)
]
```

---

## 6. ベースラインモデル

### 6.1 2.5D axial slice MIL

まず、以下のような椎体分類モデルを構築する。

```text
1椎体のaxialスライス列
      ↓
各スライスを2.5D入力としてCNNへ入力
      ↓
スライスごとの特徴量を抽出
      ↓
スライスごとの骨折スコアを出力
      ↓
top-k poolingで椎体スコアへ集約
      ↓
椎体骨折あり/なしを分類
```

### 6.2 出力

各スライスに対して骨折スコアを出力する。

```text
p_slice[z]
```

椎体全体の骨折スコアは、スライススコアを集約して算出する。

```text
p_vertebra = top-k mean(p_slice)
```

max poolingでは1枚のスライスのノイズに影響されやすいため、最初はtop-k meanを用いる。

例：

```text
p_vertebra = mean(top3 slice scores)
```

---

## 7. スライスラベルの利用

本研究では、椎体ラベルを主教師、スライスラベルを補助教師として扱う。

### 7.1 椎体ラベル

椎体ラベルは信頼できる主教師として用いる。

```text
L_vertebra = BCE(p_vertebra, y_vertebra)
```

### 7.2 スライスラベル

スライスラベルは完全に正確であるとは限らないため、補助的に利用する。

単純に各スライスへBCEをかけるのではなく、陽性範囲内のtop-kスコアを陽性に近づける形にする。

```text
p_pos_range = top-k mean(p_slice in annotated positive range)
L_pos_range = BCE(p_pos_range, 1)
```

一方で、陽性範囲から十分離れたスライスは陰性として扱う。

```text
L_neg_slice = BCE(p_slice outside positive range, 0)
```

陽性範囲の境界付近はラベルのズレが生じやすいため、ignore領域とする。

例：

```text
annotated positive range: 30〜35

ignore:
  28〜29
  36〜37

positive:
  30〜35

negative:
  その他
```

### 7.3 損失関数

全体の損失は以下のようにする。

```text
Loss =
  L_vertebra
  + λ1 L_pos_range
  + λ2 L_neg_slice
```

最初の設定例：

```text
λ1 = 0.2
λ2 = 0.2
```

スライスラベルが不完全であるため、スライス損失の重みは小さめから開始する。

---

## 8. 4領域分割の利用方法

### 8.1 単純な入力追加ではなく、出力解析に使う

4領域マスクを単純に入力へ追加しても、椎体分類精度の向上には直結しない可能性がある。したがって、まずは学習済み分類モデルの出力を4領域ごとに集計する。

具体的には、以下のいずれかを4領域マスクで集計する。

* Grad-CAM
* attention map
* feature map
* slice score
* slice-level heatmap

### 8.2 領域別スコアの算出

各領域におけるスコアを以下のように算出する。

```text
region_score[r] = mean(score_map inside region r)
```

または、

```text
region_score[r] = top-k mean(score_map inside region r)
```

対象領域は以下の4つである。

```text
r ∈ {
  left_TF,
  right_TF,
  body,
  posterior
}
```

これにより、椎体分類モデルの出力を以下のように分解できる。

```text
C5 fracture probability: 0.86

left_TF score: 0.12
right_TF score: 0.74
body score: 0.35
posterior score: 0.21
```

この場合、モデルはC5を骨折陽性と判断し、その中でも右横突孔領域に高い骨折疑いを示していると解釈できる。

## 9. 発展モデル：Region-aware MIL

ベースラインの後に、4領域マスクを明示的にモデル構造へ組み込む。

### 9.1 概要

CNNで得られたfeature mapに対して、4領域マスクを用いたmasked poolingを行う。

```text
feature map
   ↓
left_TF mask pooling
right_TF mask pooling
body mask pooling
posterior mask pooling
   ↓
4つのregion feature
   ↓
4つのregion score
```

### 9.2 出力

各スライス、各領域に対して骨折スコアを出力する。

```text
p[z, r]
```

例：

```text
p[z, left_TF]
p[z, right_TF]
p[z, body]
p[z, posterior]
```

椎体全体のスコアは、すべてのスライス・領域スコアから集約する。

```text
p_vertebra = top-k mean over p[z, r]
```

### 9.3 利点

このモデルでは、4領域分割を単なる入力ではなく、領域別スコアを出力するための構造として利用できる。

そのため、分類精度の向上だけでなく、以下の臨床的に意味のある出力が得られる。

* どのスライスが疑わしいか
* どの領域が疑わしいか
* 横突孔領域に骨折が及んでいる可能性があるか
* VAIリスク評価につながる所見があるか

## 10. 実験計画

### Experiment 1: 椎体分類ベースライン

#### 目的

椎体単位の骨折分類がどの程度可能かを確認する。

#### 入力

```text
CT 2.5D slices
```

または、

```text
CT 2.5D slices + vertebra mask
```

#### 出力

```text
p_vertebra
```

#### 評価指標

* AUROC
* AUPRC
* sensitivity
* specificity
* F1 score
* fold間のばらつき
* C1, C2, C3〜C7別の性能

---

### Experiment 2: スライス補助ラベルの効果検証

#### 目的

不完全なスライスラベルが椎体分類およびスライススコア学習に有効かを検証する。

#### 比較

```text
A: L_vertebra only
B: L_vertebra + λ L_slice BCE
C: L_vertebra + λ1 L_pos_range + λ2 L_neg_slice
```

#### 評価

* 椎体分類性能
* 陽性スライス範囲でスコアが高くなるか
* 陰性スライスでスコアが抑制されるか
* スライススコアの可視化

---

### Experiment 3: 4領域マスクによる出力解析

#### 目的

分類モデルが注目した場所を4領域ごとに分解し、領域別骨折疑いスコアを算出する。

#### 方法

学習済みモデルから以下を取得する。

* slice score
* Grad-CAM
* attention map
* feature map

それらを4領域マスクで集計する。

```text
region_score[r] = top-k mean(score_map inside region r)
```

#### 評価

* 骨折陽性椎体で高スコア領域が妥当か
* 横突孔領域に高スコアが出る症例があるか
* 医師による可視化評価が可能か

---

### Experiment 4: Region-aware MIL

#### 目的

4領域マスクを明示的にモデル構造へ組み込み、slice-region scoreを出力する。

#### 入力

```text
CT 2.5D slices
+ vertebra mask
+ 4-region masks
```

#### 出力

```text
p[z, left_TF]
p[z, right_TF]
p[z, body]
p[z, posterior]
```

#### 集約

```text
p_vertebra = top-k mean over z and r
```

#### 評価

* 椎体分類性能
* 領域別スコアの解釈性
* 横突孔領域スコアの妥当性
* 少数領域ラベルがある場合はRegion Hit@1 / Hit@2

---

## 11. 評価方針

### 11.1 椎体分類評価

主評価は椎体単位の骨折分類性能とする。

```text
AUROC
AUPRC
Sensitivity
Specificity
F1 score
```

不均衡データであることを考慮し、AUROCだけでなくAUPRCとsensitivityを重視する。

### 11.2 スライス局在評価

スライスラベルがある場合、以下を評価する。

```text
positive range内のtop-kスコア
negative sliceでの平均スコア
slice-level AUPRC
peak slice distance
```

ただし、スライスラベルは完全な正解ではないため、過度に厳密な評価は避ける。

### 11.3 領域別評価

領域別正解ラベルがない場合、まずは可視化評価を行う。

可能であれば、骨折陽性椎体の一部に対して医師が以下を確認する。

```text
左横突孔領域に骨折あり
右横突孔領域に骨折あり
椎体中心部に骨折あり
後方要素に骨折あり
複数領域にまたがる
```

その場合、以下の指標を用いる。

```text
Region Hit@1
Region Hit@2
TF involvement sensitivity
TF involvement specificity
```

## 12. 期待される結果

本研究では、4領域マスクを単純に入力へ追加するだけでは、椎体分類精度が大きく向上しない可能性がある。

しかし、4領域マスクを用いて分類モデルの出力を解剖学的領域ごとに分解することで、以下のような臨床的に有用な情報を提示できる可能性がある。

* 骨折が疑われるスライス
* 骨折が疑われる解剖学的領域
* 横突孔領域への骨折波及の可能性
* VAIリスク評価につながる補助情報

これにより、従来の椎体単位分類モデルでは得られなかった「どこが折れているか」という情報を提示できる。

---

## 13. 本研究の新規性

本研究の新規性は、頸椎骨折検出において、単なるC1〜C7の椎体単位分類ではなく、解剖学的4領域分割を用いて骨折リスクを領域別に提示する点にある。

特に、横突孔領域を左右に分けて評価することで、椎骨動脈損傷リスクと関係する可能性のある骨折所見を明示的に提示できる。

また、完全な領域別骨折アノテーションが存在しない状況でも、椎体ラベルと不完全なスライスラベルを組み合わせた弱教師あり学習により、スライスおよび領域レベルの骨折疑いスコアを推定する点も特徴である。

## 14. 今後の実装順序

### Step 1: 2.5D axial MIL baseline

```text
Input:
  CT[z-2:z+2]
  vertebra_mask[z]

Output:
  p_slice[z]

Aggregation:
  top-k mean

Loss:
  L_vertebra
```

### Step 2: スライス補助ラベルの追加

```text
Loss =
  L_vertebra
  + λ1 L_pos_range
  + λ2 L_neg_slice
```

### Step 3: 4領域マスクによる後処理解析

```text
slice score / CAM / attention map
  ↓
4-region masks
  ↓
region scores
```

### Step 4: Region-aware MIL

```text
CNN feature map
  ↓
masked region pooling
  ↓
slice-region scores
  ↓
top-k aggregation
  ↓
vertebra score
```

### Step 5: 少数領域ラベルによる妥当性検証

骨折陽性椎体の一部について、医師に4領域別骨折ラベルを確認してもらい、モデルの領域スコアが妥当か評価する。

---

## 15. まとめ

本研究では、まず2.5D axial slice MILによる椎体単位の骨折分類ベースラインを構築する。その上で、不完全なスライスラベルを補助教師として利用し、スライスごとの骨折スコアを学習する。

次に、4領域分割マスクを用いてモデル出力を解剖学的領域ごとに分解し、左横突孔、右横突孔、椎体中心部、後方要素の領域別骨折疑いスコアを算出する。

最終的には、横突孔領域に高い骨折スコアが認められる症例をVAIリスクが高い可能性のある症例として提示し、臨床的な意思決定支援につなげることを目指す。
